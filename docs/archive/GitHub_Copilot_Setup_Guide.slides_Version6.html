Introduction

Welcome! This guide is for any VA employee—not just IT pros. It walks you through getting set up with GitHub Copilot Enterprise, VS Code, all required extensions, and your workspace. Every step is plain English. No coding experience required!

What is Copilot & Why Use It?

  • GitHub Copilot is your digital assistant: it helps you write and edit, automate chores, organize files, and more.
  • It can read your files (never add patient or personal info!), help format and summarize, and automate repetitive tasks.
  • Saves hours every week for all staff—clinical, admin, research, and more.

Before You Start: Prerequisites & Security

  • Valid VA.gov email address (no personal emails!)
  • Permission to install software (ask IT if unsure)
  • Access to OneDrive or a VA network drive to save your work
  • Basic Windows skills (open folders, right-click, etc.)
Security Reminders:
  • Never put patient or personal info (PHI/PII) in Copilot workspaces or chats
  • Only use your VA.gov email for GitHub
  • Save your GitHub password somewhere safe (no PIV login)
  • Ask for help if you’re stuck!

Phase 1: Get Your GitHub Account & Access

1Create Your GitHub Account

  1. Go to github.com/signup
  2. Register with your VA.gov email and a strong password
  3. Pick a professional username (visible to others)
  4. Check your inbox and verify your email
Tip: If your preferred username is taken, try adding your VA ID or department.

2Request Access to VA’s GitHub

  1. Go to VA OIT GitHub Services (VPN required offsite)
  2. Complete the "Individual Access Request" with your name, VA.gov email, and new GitHub username
  3. Submit and wait for approval (usually 24-36 hours)
You’ll get an email when approved. Don’t move to Step 3 until then!

3Confirm You Have Copilot Access

  • Visit GitHub Copilot Features
  • Look for: You are assigned a seat as part of a GitHub Copilot Enterprise subscription managed by department-of-veterans-affairs
  • Test Copilot Web: github.com/copilot
If you don’t see this after 48 hours, email DevOpsTeam@va.gov

Phase 2: Set Up Your Work Environment

These steps unlock Copilot’s full “Agent Mode.” Highly recommended!

4Install Visual Studio Code (VS Code)

  1. Download from code.visualstudio.com/download
  2. Select “Windows x64 User Installer” (for most VA computers)
  3. Run the installer, accept defaults, check “Add to PATH” and “Create desktop icon” if offered
VS Code is free, easy, and built for Copilot.

5Create Your Workspace Directory

  1. Open VS Code
  2. Go to FILE → OPEN FOLDER
  3. Choose a location (OneDrive recommended): C:\Users\%USERNAME%\OneDrive\GitHub_Workspace\
  4. Create a new folder named GitHub_Workspace and select it

6Add Other Work Folders

  1. In VS Code: FILE → ADD FOLDER TO WORKSPACE
  2. Add folders you use often (Desktop, Documents, Team drive, etc.)
Security: Never add folders with PHI/PII or sensitive info—Copilot can read all workspace folders.

7Save Your Workspace

  1. Go to FILE → SAVE WORKSPACE AS
  2. Save as GitHub_Workspace.code-workspace in your GitHub_Workspace folder

8Make a Desktop Shortcut

  1. Find your workspace file in File Explorer
  2. Right-click → Send to → Desktop (Create Shortcut)
  3. (Optional) Rename to “GitHub Copilot Workspace”
Always open VS Code with this shortcut!

9Install Must-Have Extensions

  1. Open your VS Code Workspace
  2. Press Ctrl+Shift+X to open Extensions panel
  3. Search and install each of these:
    • GitHub Copilot (github.copilot)
    • GitHub Copilot Chat (github.copilot-chat)
    • Copilot Mermaid Diagrams (ms-vscode.copilot-mermaid-diagram)
    • Web Search for Copilot (ms-vscode.vscode-websearchforcopilot)
    • Vision for Copilot Preview (ms-vscode.vscode-copilot-vision)
    • VS Code Icons (vscode-icons-team.vscode-icons)
    • PDF Viewer (tomoki1207.pdf)
    • Excel Viewer (grapecity.gc-excelviewer)
Tip: Only install extensions from trusted publishers and with good ratings!

Phase 3: Copilot Configuration & Personalization

10Open Copilot Chat

  • Press Ctrl+Alt+I (sign in to GitHub if prompted)
  • Try asking: “What files do you see in my workspace?” or “What can you help me with?”

11Configure Copilot for Your Needs

  1. Paste this in Copilot Chat:
    I want you to configure my VS Code settings, workspace settings, systems, and yourself to help me permanently improve my projects and make my work the most efficient. Look at my active workspace files, folders, and content to get an idea of the kind of work I do. Ask me questions that I can answer which will guide you to make recommended changes to any settings, connections, defaults, or preferences that will improve the quality of my work going forward. Put special focus and emphasis on questions related to the type of projects I work on and subjects contained. Collect all these questions and answers, then modify my VS Code global settings.json file to permanently save all these approved changes. Do not actually save any changes or modifications without explicitly asking me first.
  2. Answer Copilot’s questions about your work style, file types, favorite settings, etc.
  3. Review and approve any recommended changes before they’re made.

12Generate Custom Copilot Instructions

  1. Click the GEAR icon (⚙️) in Copilot Chat
  2. Select GENERATE INSTRUCTIONS
  3. This creates a copilot-instructions.md file
  4. Use this prompt for deeper personalization:
    I want you to perform a full active workspace analysis and evaluate the type of projects that I routinely work on. Use any context and analysis information to modify my copilot-instructions.md or copilot prompt files, and add as many descriptive, recommended, helpful, and impactful updates or changes to those files that will help me perform my work more efficiently. Then, I want you to generate a series of 25 questions that you will ask me one-by-one in an interactive question & answer session, with each answer I give you helping you expand and improve all the context and references in these instructional or prompt files all with the goal of increasing my productivity and expanding my ability to take advantage of using Copilot Agent going forward.
  5. Answer the questions—the more you share, the more helpful Copilot becomes!

13Personalize Your Experience

  • Tell Copilot your name, job title, and department
  • Ask it to use VA colors, fonts, and styles in documents
  • Request templates for common tasks (emails, reports, agendas)
  • Remind Copilot to always flag PHI/PII risks

Phase 4: Real-World Examples & Use Cases

What can Copilot do for you?

  • Documentation: Summarize policies, create training materials, generate presentations
  • Data Analysis: Clean up spreadsheets, make charts, automate reports
  • Automation: Script repetitive tasks, organize folders, send reminders
  • Communication: Draft emails, build checklists, create meeting agendas
  • Education: Build quizzes, flashcards, or learning modules for staff/patients
  • Healthcare: Update clinical protocols, generate patient education, manage QI projects
Ask Copilot: “Give me examples of how you can help with [your daily task].”

Troubleshooting & Support

  • Copilot not working? Check you’re signed in with VA account, restart VS Code, or wait for enterprise access
  • Extensions not installing? Try running VS Code as admin, check for typos in extension names
  • Workspace problems? Always open from your .code-workspace shortcut
